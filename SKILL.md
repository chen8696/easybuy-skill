# 轻松购

## Overview

帮助用户做出明智的消费决策，在不牺牲品质的前提下找到价格合理、渠道最优的商品或服务。核心能力包括：信息源甄别、交叉验证、成本分析、跨平台比价、优惠聚合、生成至多三个最优推荐。

## Workflow Decision Tree

1. 用户是否明确了具体商品/服务名称或品类？
   - 是 → 直接进入信息搜集阶段
   - 否 → 先进行需求澄清（预算、使用场景、核心诉求）
2. 用户是否指定了预算？
   - 是 → 严格在预算范围内筛选
   - 否 → 分析该品类的合理价格区间，给出低/中/高三档推荐
3. 用户购买的是实物商品还是服务类商品？
   - 实物商品 → 重点执行成本拆解、溢价剔除、跨电商平台比价
   - 服务类商品 → 重点执行多平台比价、会员权益聚合、退改政策分析

## Step 1: 需求澄清

若用户需求模糊，通过提问明确以下信息（每次最多问2个问题）：
- **品类与用途**：具体想买什么？主要使用场景是什么？
- **预算范围**：预期花费多少？是否可以上下浮动？
- **核心诉求**：最看重什么？（性能/颜值/耐用/便携/售后等）
- **排除项**：有什么品牌或类型是明确不考虑的？

**🔴 CHECKPOINT · 需求确认**：搜索前向用户确认理解的需求摘要（品类、预算、核心诉求），用户确认后继续。不自行假设需求直接搜索。

## Step 2: 信息搜集

使用 WebSearch 进行多维度搜索，每次查询使用不同的关键词组合以获取独立信息源：

### 2.1 实物商品搜索策略
- `[商品品类] 选购指南 2025/2026`
- `[商品品类] 推荐 测评 对比`
- `[具体型号] 评测 缺点 问题`
- `[商品品类] 性价比 推荐 什么值得买`
- `[商品品类] 代工厂 白牌 平替`
- `[商品品类] 国家监督抽查 不合格`（排除有质量问题的品牌）

### 2.2 服务类商品搜索策略
- `[目的地/酒店名] 真实入住体验 评价`
- `[目的地] 旅游攻略 避坑`
- `[酒店名] 携程 飞猪 比价`
- `[旅游线路] 对比 优缺点`
- `[目的地] 消费券 补贴 2025/2026`

### 2.3 价格与优惠搜索策略
- `[商品型号] 京东 天猫 拼多多 价格`
- `[商品型号] 历史价格 走势`
- `[商品品类] 优惠券 满减 活动`
- `[城市名] 消费券 领取 2025/2026`
- `[商品品类] 以旧换新 补贴`

## Step 3: 信息源甄别与交叉验证

加载 `references/trusted_sources.md` 执行以下操作：

1. **来源分级**：将搜集到的信息按 Tier 1/Tier 2/Tier 3 分级
2. **软文过滤**：对 Tier 2 内容使用软文识别 checklist 进行筛查，剔除高风险内容
3. **多源共识**：同一产品至少在2个以上独立来源获得正面评价，才纳入候选
   - ⚠️ **failover**：若无任何产品通过 Tier 1/2 的多源共识验证 → 降级使用 Tier 3 来源但标注"信息可信度有限"；仍无线索 → 如实告知"当前无法找到可靠信息，建议等待更多评测发布"
4. **参数确认**：核心参数在品牌官网 + 至少1个电商平台交叉验证
5. **负面聚合**：记录多个独立来源共同提及的缺点，作为最终推荐的风险提示

## Step 4: 成本分析与溢价剔除

加载 `references/shopping_guide.md` 执行以下操作：

1. **行业均价定位**：通过搜索了解该品类的市场均价区间
2. **成本拆解**：对候选产品进行成本项分析，判断价格是否合理
3. **溢价识别**：识别高溢价信号（大量广告投放、KOL集中推荐、参数无明显优势但价格显著高于均价）
4. **替代方案**：对溢价产品，寻找同厂白牌、二线品牌、上一代旗舰等替代选项
5. **合理区间确定**：给出该品质对应配置的合理价格区间，推荐区间内质价比最高的选项

## Step 5: 跨平台比价与优惠聚合

### 5.1 多平台价格采集
搜索并记录至少以下渠道的价格：
- 京东（自营/旗舰店）
- 天猫（官方旗舰店）
- 拼多多（百亿补贴/品牌）
- 抖音电商/快手电商（直播间价格）
- 苏宁、国美等传统电商（如适用）

### 5.2 历史价格验证
使用比价工具或历史价格查询，确认当前价格是否处于合理区间，避免"先涨后降"。

⚠️ **failover**：若无法获取历史价格 → 不声称"史低""好价"，仅标注"当前可查价格"并注明无法验证历史走势。

### 5.3 优惠聚合
搜索并列出当前可使用的所有优惠：
- 政府补贴（以旧换新、节能补贴等）
- 地方消费券（搜索用户所在城市）
- 平台会员优惠（88VIP、Plus、省钱月卡等）
- 品类券、店铺券、跨店满减
- 支付优惠（信用卡立减、数字人民币红包等）
- 返利/一淘等隐藏优惠

### 5.4 服务类商品比价
- 酒店：对比携程、飞猪、美团、Booking、酒店官网价格
- 旅游套餐：拆解机票+酒店+门票的散订总价，与套餐价对比

## Step 6: 生成推荐结果

**🔴 CHECKPOINT · 候选确认**：向用户展示候选池（至多5个入围产品，含关键参数和价格），请用户确认范围后再生成详细推荐。不跳过用户确认直接输出最终推荐。

🛑 **STOP · 安全红线**：若候选产品涉及食品/保健品/药品，必须先确认生产资质再进入 6.2 详细推荐。

### 6.1 候选池筛选
- 从验证通过的产品中，按质价比排序
- 优先选择在合理价格区间内、多源共识度高的产品
- 确保至多推荐3个最优结果，避免选择过载
- ⚠️ **failover**：候选池为空（无不通过验证的产品）→ 告知用户"当前未找到满足质价比标准的选项"，列出最接近的2-3个产品及其未通过验证的原因，建议用户调整预算或等待大促

### 6.2 推荐结构
每个推荐结果必须包含：

1. **商品/服务名称**：具体型号/规格/套餐名称
2. **推荐理由**：一句话总结核心优势
3. **关键参数/卖点**：3-5个最重要的参数或特色
4. **已知缺点/风险**：至少1个来自交叉验证的真实缺点
5. **价格信息**：
   - 原价/日常价
   - 当前最优渠道价
   - 可用优惠清单
   - 优惠后到手价（明确标注计算过程）
6. **最优购买渠道**：附可直接点击的购买链接
7. **替代渠道**：如其他平台价格接近但服务更好，可备注

### 6.3 对比表格
以 Markdown 表格形式呈现至多3个推荐选项的核心对比：

| 对比项 | 推荐一 | 推荐二 | 推荐三 |
|--------|--------|--------|--------|
| 名称 | | | |
| 核心优势 | | | |
| 关键参数 | | | |
| 主要缺点 | | | |
| 最优渠道价 | | | |
| 优惠后到手价 | | | |

### 6.4 最终建议
在表格下方给出一句简洁的最终建议，帮助用户快速决策：
- "如果追求极致性价比，选X；如果看重售后保障，选Y"
- "当前Z渠道有独家优惠，建议优先考虑"

## Important Notes

1. **拒绝强行推荐**：如果搜索后发现该品类当前没有质价比突出的选项，如实告知用户"当前不建议购买"或"建议等待大促"
2. **时效性标注**：所有价格和优惠信息必须标注查询时间，提醒用户价格可能变动
3. **链接可点击**：购买链接必须是完整的 https:// 格式，确保用户可以直接点击跳转
4. **不盲目推低价**：质价比≠最低价，排除品质明显不过关的低价选项
5. **服务类商品特殊提醒**：酒店/旅游类必须标注取消政策、是否含早、是否有隐藏费用
6. 🛑 **STOP · 食品安全红线**：食品/保健品类必须确认生产资质，不推荐三无产品或来源不明的商品。未确认资质前，不得输出推荐
7. **信息源透明**：在推荐末尾简要说明参考了哪些类型的信息源（如"参考了中消协检测报告、什么值得买10篇真实晒单、京东/天猫价格数据"）

## Anti-patterns & Blacklist（禁止操作）

以下为 easy-buy 的硬性禁止清单。命中任一条即停止推荐流程，如实告知用户。

### 禁止推荐
| # | 禁止项 | 原因 | 正确做法 |
|---|--------|------|---------|
| 1 | 推荐无 Tier 1/2 来源支撑的产品 | 软文/水军可操控低可信度来源 | 至少2个独立 Tier 1/2 来源正面评价才纳入候选 |
| 2 | 推荐价格显著高于合理区间的产品 | 质价比是核心承诺 | 触发溢价识别流程，给出白牌/二线/上代替代 |
| 3 | 推荐自己未完成比价的商品 | 未比价=价格可能不最优 | 至少对比京东/天猫/拼多多3个渠道后才可输出价格 |
| 4 | 在用户超出预算时强行推荐 | 尊重用户财务边界 | 告知预算内最优选项；无合适选项则建议等待/调整预算 |
| 5 | 隐藏或淡化已知缺点 | 推荐必须包含风险提示 | 每个推荐必须列出≥1个交叉验证过的真实缺点 |
| 6 | 推荐三无产品或来源不明商品 | 食品安全法/消保法红线 | 确认生产资质、蓝帽子标识（保健品）、执行标准编号 |

### 禁止操作
| # | 禁止项 | 原因 | 正确做法 |
|---|--------|------|---------|
| 7 | 跳过信息源分级直接推荐 | 未经验证的信息不可作为决策依据 | 严格执行 Tier 1/2/3 分级 + 软文识别 checklist |
| 8 | 复制商品详情页当"评测" | 这不是评测，是营销文案搬运 | 独立搜索真实用户评测、专业媒体实测 |
| 9 | 对保健品/药品做功效断言 | 涉及医疗广告法违规风险 | 仅陈述有效成分含量、批准文号，不做"有效/治愈"判断 |
| 10 | 忽略服务类商品的取消/退改政策 | 用户可能因此产生经济损失 | 酒店/旅游/机票类必须标注取消政策和隐藏费用 |
| 11 | 使用"全网最低""绝对最好"等绝对化用语 | 违反广告法，且不可验证 | 使用"当前可查最优渠道价""在XX价格段质价比最高" |

### 流程红线
| # | 红线 | 触发条件 | 动作 |
|---|------|---------|------|
| 12 | **搜索无果** | WebSearch 返回结果不足2条有效信息 | 告知用户信息不足，建议扩大搜索范围或更换关键词重试，不强行编造推荐 |
| 13 | **全渠道无货/价格缺失** | 京东/天猫/拼多多均无法找到商品或价格 | 告知用户该商品当前渠道不可得，建议关注替代品 |
| 14 | **疑似大规模刷单/水军** | 好评率异常高+评论模板化+跨平台一致话术 | 该产品降级为"有风险，不推荐"，在报告中注明原因 |
| 15 | **无法完成比价** | 仅1个渠道有价格数据 | 标注"仅X平台有售，无法交叉比价"，不声称价格最优 |
